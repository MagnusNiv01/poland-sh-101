# Repository Instructions

## README Maintenance

Before every push, check whether the user-visible behavior, setup steps, architecture, limitations, or roadmap changed.

If any of those changed, update `README.md` in the same work session before pushing. Do not push changes that make the README stale.

At minimum, verify these README sections before push:

- Current features
- Installation/development/build instructions
- Audio engine description
- Placeholder controls
- Known limitations
- Roadmap

If the README is already accurate, mention that it was checked before pushing.

## Project Constraints

- Keep the visible UI name as “Poland SH-101”.
- Do not use official Roland logos or assets.
- Do not claim this is an official Roland product.
- Keep code comments in English.
- Do not redesign the UI unless explicitly requested.
- Do not change public patch parameter names unless explicitly required.
